How the AD Downloader works

Where does the data come from?

The tool queries the Federal Register's official public API — the same database the FAA publishes all Airworthiness Directives to. This is the authoritative source; it is not scraping a website or relying on a third-party database.

How does it avoid missing ADs?

The search is deliberately broad first, then filtered using a two-stage process:

  • Stage 1 — Cast a wide net via the API: The tool asks the Federal Register for every Rule published by the FAA that mentions both the aircraft make (e.g. "Boeing") and the aircraft type (e.g. "737") since the chosen date. It retrieves up to 100 results per page and automatically loops through all pages, so nothing is truncated.
  • Stage 2 — Confirm applicability by reading the actual AD text: For every candidate returned, the tool fetches the full text of the AD and reads the (c) Applicability paragraph — the legally binding section that defines exactly which aircraft the AD applies to. It checks that both the aircraft type and the model/series appear independently in that paragraph. An AD only makes the final list if both terms are found there.

Why is the two-stage approach important?

The broad API search ensures ADs are not excluded due to unusual title wording. The applicability check ensures ADs that merely reference the target model in passing (e.g. a 737-800 AD that mentions the -400 as context) are not included. Every AD in the output genuinely applies to the specified aircraft.

What is in the output?

Each matched AD includes its AD number, ATA system code, subject description, the full applicability text, the unsafe condition description, and a direct link to the official PDF — all traceable back to the Federal Register.
